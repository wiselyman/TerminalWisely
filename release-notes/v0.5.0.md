# v0.5.0 — Find 与服务器资源

## 新增

### Find 文件搜索
- 工作区**右侧贴边工具栏**（任务管理器下方）→ 点击打开右侧抽屉
- 在当前激活页签对应机器上执行 Linux **`find`** 命令（SSH 远程；本地 Unix/macOS）
- 从**当前工作目录**搜索；可配置 `-name` 模式、`-type`、`-maxdepth`、`-iname`
- 结果列表：点击目录 `cd` 进入，点击文件打开预览
- 快捷键 **Ctrl/Cmd+F** 打开 Find 面板

### 服务器资源检测
- 贴边工具栏**第三枚图标** → 打开资源监控抽屉
- 数据范围：**当前激活页签**对应机器（本地或 SSH）
- 展示：hostname、OS/内核/架构、CPU/内存/Swap gauge、负载、运行时间、进程数
- 网络上行/下行速率（采样差值）与累计流量
- 登录用户列表、磁盘占用 progress bar
- CPU/内存 sparkline 趋势（约 1 分钟窗口）；系统信息置顶；CPU/内存/网络合为统一指标块
- 打开时 **2 秒**自动刷新

### 连接体验
- **乐观页签**：点击 SSH/本地连接后立即出现页签，后台异步建立会话
- 连接中显示 spinner 遮罩；就绪后等待首条输出前显示启动提示
- 连接中页签脉动指示；可随时关闭取消连接

## 已知限制

- 本地 **Windows** 会话暂不支持 find，请使用 SSH 连接 Linux 主机
- Find 单次最多 500 条结果
- 网络速率首帧无数据，需第二次采样
- SSH 资源/find 依赖 `/proc`、`df`、`who` 等；最小化容器可能缺字段
- 任务管理器 / Find / 服务器资源 三个抽屉**互斥**
